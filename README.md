# 🏛️ Nepal Constitution Chatbot — Phase 1

A locally-running, AI-powered chatbot that lets you ask questions about the **Constitution of Nepal (2015)** in plain English. Built with a full RAG (Retrieval-Augmented Generation) pipeline — from PDF ingestion to conversational Q&A — running entirely on your machine with no external API calls.

---

## 📸 Demo

> Ask: *"What are the fundamental rights of citizens?"*
> The chatbot retrieves the most relevant constitutional articles and generates a cited, grounded answer like:
> *"Citizens are guaranteed freedom of expression (Article 17) and the right to equality before the law (Article 18)..."*

---

## 🧠 How It Works

This project is a 3-stage RAG pipeline:

```
📄 PDF  ──►  [ingest.py]  ──►  articles.json
                                     │
                                     ▼
                             [embed.py]  ──►  ChromaDB (Vector Store)
                                                      │
                                                      ▼
                                User Query  ──►  [app.py]  ──►  Answer with Citations
```

### Stage 1 — Ingestion (`Script/ingest.py`)
- Opens the Constitution of Nepal PDF using **PyMuPDF (`fitz`)**
- Extracts raw text from every page
- Splits the text into individual articles using regex pattern matching
- Cleans and normalizes the text
- Saves all articles as structured JSON (`articles.json`)

### Stage 2 — Embedding (`Script/embed.py`)
- Loads articles from `articles.json`
- Encodes each article using **`intfloat/multilingual-e5-base`** — a multilingual sentence embedding model
- Stores the embeddings in a persistent **ChromaDB** vector database with cosine similarity
- Processes articles in batches of 32 for efficiency

### Stage 3 — Retrieval & Generation (`app.py`)
- Takes a user's question and embeds it using the same E5 model
- Queries ChromaDB to retrieve the **top 5 most relevant articles** (ranked by cosine similarity)
- Builds a strict RAG prompt that instructs the LLM to answer **only from retrieved articles**, with citations
- Sends the prompt to a local **Ollama** LLM (`qwen3:8b`) for answer generation
- Displays the answer with article references and a relevance score for each retrieved article

## ⚙️ Setup & Installation

### Prerequisites

Before starting, make sure you have:

- Python 3.10 or higher
- [Ollama](https://ollama.com/) installed and running
- The **Constitution of Nepal 2015 PDF** (place it in the `data/` folder as `constitution_nepal_2015.pdf`)

### 1. Clone the repository

```bash
git clone https://github.com/Utkrista123/Legal_Chatbot_Phase_1.git
cd Legal_Chatbot_Phase_1
```

### 2. Install dependencies

```bash
python -m pip install -r requirement.txt
```

> ⚠️ Use `python -m pip` (not just `pip`) to ensure packages install into the correct Python environment.

### 3. Pull the LLM model via Ollama

```bash
ollama pull qwen3:8b
```

Make sure Ollama is running in the background:

```bash
ollama serve
```

### 4. Run Stage 1 — Ingest the PDF

```bash
python Script/ingest.py
```

This reads `data/constitution_nepal_2015.pdf`, splits it into articles, and saves `json/articles.json`.

Expected output:
```
...processed 20 pages
...processed 40 pages
Extracted: 185432 characters
Found 308 articles.
Article 1: This Constitution of Nepal made and promulgated by the Constituent Assembly...
```

### 5. Run Stage 2 — Generate Embeddings

```bash
python Script/embed.py
```

This embeds all articles and stores them in ChromaDB under `chroma_db/`.

Expected output:
```
Loaded 308 articles.
Batch 1/10 - articles 1-32 embedded
Batch 2/10 - articles 33-64 embedded
...
===================================================
  DONE. Database saved to ./chroma_db/
  Next step: run streamlit run app.py
===================================================
```

> ⚠️ You only need to run Stages 1 and 2 **once**. The ChromaDB database persists between sessions.

### 6. Run Stage 3 — Launch the Chatbot

```bash
streamlit run app.py
```

Open your browser at `http://localhost:8501` and start asking questions!

## 🔍 Key Design Decisions

**Why `intfloat/multilingual-e5-base`?**
This model is specifically trained for retrieval tasks. It requires `passage:` prefix for indexed documents and `query:` prefix for search queries — this project correctly implements both, which significantly improves retrieval accuracy.

**Why Ollama + Qwen3:8B?**
The entire pipeline runs locally — no data leaves your machine, no API keys required, no cost per query. This is important for legal use cases where data privacy matters.

**Why strict citation prompting?**
The system prompt enforces that the LLM answers *only* from the retrieved articles and cites every sentence. This minimizes hallucination, which is critical for legal information.

**Relevance Score Formula:**
```python
relevance = round((1 - distance / 2) * 100, 1)
```
Converts ChromaDB's cosine distance (0–2) into a human-readable 0–100% relevance score.

## 🚧 Known Limitations

- Only covers the **English translation** of the Constitution of Nepal (2015)
- The article splitting regex may occasionally misparse complex article numbering — review `articles.json` to verify
- No conversation memory — each question is answered independently without context from prior messages
- Performance depends on your hardware; `qwen3:8b` requires ~8GB of RAM
